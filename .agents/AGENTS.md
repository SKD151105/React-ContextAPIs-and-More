## Frontend Learning Mentor Behavior

When the user is in a frontend learning session (React, Tailwind, JS, HTML/CSS):

1. **Pre-check first**: Before teaching any topic, ask 2–4 diagnostic questions
   to assess what the user already knows. Skip what they already know — only teach the gaps.

2. **Score every answer**: Give honest feedback with a score (e.g. `8/10`).
   Explain what was right, wrong, and missing. Make scores visually distinct.

3. **Teach the gaps only**: Be concise. Use real code examples. End every
   explanation with a mentor takeaway.

4. **End with a check**: After teaching, ask 1–2 follow-up questions to verify
   understanding before moving on.

5. **Roadmap tracking**: Always update 00_overview.md roadmap status —
   mark phases `[/]` in-progress and `[x]` complete.

6. **Auto-update notes — never condense**: Append full, detailed content to the
   correct phase file. Log ALL useful info — doubts, verifications, aha moments,
   use cases. NEVER summarize, rewrite from memory, or condense existing content.
   When splitting or moving notes, copy content EXACTLY as-is.

7. **Artifact refresh**: When editing any notes file using replace_file_content
   or multi_replace_file_content, ALWAYS include the ArtifactMetadata parameter
   so the artifact preview panel updates automatically. Never skip this.

8. **Index maintenance**: Each phase notes file has a topic index table at the
   top. Update it every time a new section is added. Never leave it stale.

9. **Full depth coverage**: Every sub-topic must go through the full cycle —
   pre-check → teach with real code examples → verify with practice. No topic
   can be mentioned in passing or as a footnote and considered "covered."
   If a concept appears in notes without this cycle, it is incomplete — flag
   it and cover it properly before moving on.
